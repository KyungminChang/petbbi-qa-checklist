실행 순서 (고객 세그먼트 "니즈 정면타격" 여부로 2차 정렬)
  1. 상시 게이트웨이 (7건) — TC-066/069/070(가입·로그인) → TC-001(프로필 등록) → TC-005(사진 업로드) → TC-015(캐릭터 생성) → TC-018(캐릭터 저장). 5개 세그먼트 전부가 예외 없이 거치는 경로.
  2. 조건부 게이트웨이 (2건) — TC-023(네트워크 끊김) → TC-027(AI 신뢰도 미달 폴백). 특정 조건에서만 발생해 확실성이 낮아 한 단계 뒤로.
  3. 데이터 손실 정면타격 (1건) — TC-079(30일 내 데이터 복구 실패). 펫 추모 사용자에게는 비가역적 사안이라 3번째 배치.
  4. 데이터 무결성 간접 (4건) — TC-008 → TC-077 → TC-078 → TC-081. 니즈를 지키는 "안전장치" 성격이라 후순위.
  5. 법적 전용 (1건) — TC-072(만 14세 미만 가입 차단). 어느 세그먼트 니즈와도 직접 연관이 없어 체감 우선순위상 마지막(단, P0 분류 자체는 순서와 무관하게 동일).
근거(리스크) · 실패 시 조치
TC-015(캐릭터 생성)는 일기·커뮤니티 등 하위 기능 전체의 선행 차단 지점이라 실패 시 나머지 P0 케이스 상당수도 검증 자체가 불가능해짐. 실패는 전체 15건을 다시 도는 게 아니라 실패한 케이스만 단독 핫픽스 후 재검증하되, 그룹 1(상시 게이트웨이) 소속이 실패하면 후속 케이스도 함께 재검증한다.
